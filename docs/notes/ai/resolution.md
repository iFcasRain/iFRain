## 一、理论课内容回顾

### 1.1 基本概念

#### 1.1.1 核心案例：Alpine Club 问题

**问题描述**

- Tony, Mike, John 属于阿尔卑斯俱乐部
- 俱乐部中不是滑雪者的成员，都是登山者
- 登山者不喜欢雨；不喜欢雪的人，都不是滑雪者
- Mike 讨厌所有 Tony 喜欢的东西，喜欢所有 Tony 讨厌的东西
- Tony 喜欢雨和雪
- 提问：俱乐部中是否存在是登山者但不是滑雪者的成员？

**形式化知识库 (KB)**

- 事实 (Facts)

  - `A(tony)`、`A(mike)`、`A(john)`
  - `L(tony,rain)`、`L(tony,snow)`

  

- 规则 (Rules)

  - `∀x(A(x) ∧¬S(x) → C(x)`
  - `∀x(C(x) → ¬L(x,rain)`
  - `∀x(¬L(x,snow) → ¬S(x)`
  - `∀x(L(tony,x) → ¬L(mike,x)`
  - `∀x(¬L(tony,x) → L(mike,x)`

  

**待证明结论**：`∃x(A(x) ∧C(x) ∧¬S(x)` 是否成立

#### 1.1.2 核心术语定义

1. **常量 (constant)**：指代具体实体，如俱乐部成员`tony,mike,john`、天气`rain,snow`

2. **变量 (variable)**：指代未知量，如`x,y,z`等符号

3. 项 (term)

   ：递归定义的参数单元

   - 变量是项（可视为 0 元函数）
   - 若`t1,t2…tn`是项，`f`是 n 元函数，则`f(t1,t2,…,tn)`也是项
   - 🔴 Tips：一阶逻辑中谓词不是项，不能作为函数 / 谓词的参数，不存在`f(P(x)`的复合方式（二阶逻辑支持）

   

4. 谓词 (predicate)

   ：描述参数的属性或参数间的关系

   - 零元谓词：退化为命题
   - 单元谓词：仅 1 个参数，描述参数属性，如`A(x)`表示 x 属于阿尔卑斯俱乐部
   - 多元谓词：多个参数，描述参数间关系，如`L(x,y)`表示 x 喜欢 y

   

5. **事实 (fact)**：谓词中变量完成实例化后的结果，如`S(tony)`、`L(tony, rain)`

6. 规则 (rule)

   ：也叫公式，通过递归规则定义

   - 若`t1,t2…tn`是项，`P`是 n 元谓词，则`P(t1,t2,…,tn)`是原子公式
   - 若`t1,t2`是项，则`t1=t2`是原子公式
   - 若`α`和`β`是公式，则`¬α`、`α∧β`、`α∨β`、`∃α`、`∀α`都是公式
   - 🔴 Tips：`α→β`等价于`¬α ∨β`，`α↔β`等价于`(α→β) ∧(β→α)`，可通过基础符号替代，无需单独定义

   

7. **可满足性**：是否存在一组变量实例化（赋值），使得目标公式成立。Alpine Club 问题的核心，就是判断`A(x) ∧C(x) ∧¬S(x)`是否可满足

8. 逻辑蕴含与逻辑推论

   - 逻辑蕴含`S |= α`：对于任意变量赋值，若 S 为真，则 α 必为真
   - 逻辑推论`S |- α`：存在一条合规的推理路径，可从 S 出发推导证明 α

   

### 1.2 命题逻辑归结算法

#### 1.2.1 算法核心流程

1. 将待证明结论`α`取否定，加入知识库 KB 中

2. 将更新后的 KB 转换为**子句形式 (Clausal form)**，得到子句集 S

3. 反复调用单步归结规则：

   - 若得到**空子句**（记为`()`），说明`KB ∧¬α`不可满足，算法终止，可证`KB |= α`
   - 若归结至无新子句产生，且全程未得到空子句，则`KB |= α`不成立

   

#### 1.2.2 核心规则定义

1. 子句形式 (Clausal form)

   - 单个子句对应一个元组，元组内每个元素是原子公式 / 原子公式的否定，元素间为**析取关系**（任意一个原子为真，子句即为真）
   - 示例：子句`¬child∨¬male∨boy`对应`(¬child,¬male,boy)`，空子句`()`对应逻辑假`False`
   - 子句集 S 是元组的集合，子句间为**合取关系**（所有子句必须同时为真）

   

2. 单步归结规则

   - 从两个子句中，找到互为否定的相同原子（冲突对）
   - 移除该冲突原子，将两个子句的剩余部分合并为一个新子句，加入子句集 S
   - 示例：`(¬child,¬female,girl)`和`(child)`合并，得到新子句`(¬female,girl)`

   

#### 1.2.3 算法示例

**KB**：

- `FirstGrade`

- `FirstGrade → Child`

- `Child ∧ Male → Boy`

- `Kindergarten → Child`

- `Child ∧ Female → Girl`

- `Female`

  

  待证明

  ：

  `KB |= Girl`

子句集转换结果：

1. `(FirstGrade,)`
2. `(~FirstGrade, Child)`
3. `(~Child, ~Male, Boy)`
4. `(~Kindergarten, Child)`
5. `(~Child, ~Female, Girl)`
6. `(Female,)`
7. `(~Girl,)`（结论取反）

最终可通过归结推导出空子句，证明`KB |= Girl`成立。

### 1.3 MGU (最一般合一) 算法

#### 1.3.1 核心定义

1. 合一 (unifier)

   ：通过变量替换，让两个原子公式完全等价，从而使两个子句具备可归结的冲突对。

   - 示例：子句`(P(john),Q(fred),R(x)`和`(¬P(y),R(susan),R(y)`，通过将`y`替换为`john`，得到可归结的冲突对`P(john)`和`¬P(john)`

   

2. **最一般合一 (MGU)**：能让两个原子公式等价的、最简的变量替换集合，无冗余替换。

#### 1.3.2 算法规范

- **输入**：两个谓词相同、参数项不同的原子公式（字符串格式）

  

- **输出**：MGU 变量替换字典，若无合法合一则返回空字典

  

- **算法流程**：

  

  1. 初始化：`k=0`，替换集合`σ₀={}`，待合一集合`S₀={f, g}`

  2. 若`Sₖ`中的公式完全等价，返回`σₖ`作为 MGU 结果；否则找出`Sₖ`中的不匹配项`Dₖ={e1, e2}`

  3. 若

     ```
     e1=V
     ```

     是变量，且

     ```
     e2=t
     ```

     是不包含变量 V 的项：

     - 将`V=t`加入替换集合，`σₖ₊₁=σₖ ∪ {V=t}`
     - 将`Sₖ`中所有变量 V 替换为 t，得到`Sₖ₊₁`
     - `k=k+1`，回到步骤 2

     

  4. 若不满足步骤 3 的条件，合一失败，返回空字典

  

- 🔴 Tips：变量替换从原子公式中提取，最终需施加给整个子句。

  

#### 1.3.3 示例

- 调用`MGU('P(xx,a)','P(b,yy)')`，返回`{'xx':'b','yy':'a'}`
- 调用`MGU('P(a,xx,f(g(yy))','P(zz,f(zz),f(uu)')`，返回`{'zz':'a','xx':'f(a)','uu':'g(yy)'}`

### 1.4 一阶逻辑的归结算法

#### 1.4.1 算法核心流程

1. 将待证明结论`α`取否定，加入知识库 KB 中

2. 将更新后的 KB 转换为子句形式，得到子句集 S

3. 反复调用一阶逻辑单步归结规则：

   - 若得到空子句，说明`KB ∧¬α`不可满足，算法终止，可证`KB |= α`
   - 若归结至无新子句产生，且全程未得到空子句，则`KB |= α`不成立

   

#### 1.4.2 单步归结规则

1. 基于两个子句，使用 MGU 算法找到可合一的冲突原子对（原子及其否定）
2. 对两个子句应用 MGU 完成变量替换，移除冲突原子，将剩余部分合并为新子句，加入子句集 S
3. 示例：`(¬Student(x),HardWorker(x)`和`(HardWorker(sue)`，通过 MGU`{x=sue}`合并为`(¬Student(sue)`

### 2.1 通用要求

#### 2.1.1 输出格式规范

需严格按如下格式输出归结全过程：

plaintext











```
1. (P(x),Q(g(x))
2. (R(a),Q(z),~P(a)
3. R[1a,2c]{X=a} (Q(g(a),R(a),Q(z)
```

- `R`：标记本次为归结步骤
- `1a`：第 1 个子句的第 1 个原子公式（a 对应第 1 位，b 对应第 2 位，以此类推），即`P(x)`
- `2c`：第 2 个子句的第 3 个原子公式，即`~P(a)`
- `{X=a}`：本次归结使用的 MGU 变量替换规则
- 末尾为本次归结得到的新子句

#### 2.1.2 Python 数据结构规范

1. **公式（原子 / 否定原子）存储**

   

   - 用字符串存储，否定符号`¬`用`~`代替
   - 谓词首字母大写，如`A`、`Student`；参数间用逗号`,`间隔，无空格
   - 常量用小写单词 / 小写字母表示，如`a`、`tony`
   - 变量用小写字母 / 小写单词表示，本次作业无`∃`、`∀`量词符号
   - 示例：`~child`、`R(a)`、`~P(a,zz)`、`Student(tony)`

   

2. **子句存储**

   

   - 用`tuple`（元组）存储，元组内每个元素为一个公式字符串
   - 示例：`¬child∨¬male∨boy`存储为`('~child', '~male', 'boy')`；`¬S(z) ∨ L(z, snow)`存储为`('~S(z)', 'L(z,snow)')`

   

3. **子句集存储**

   

   - 用`set`（集合）存储，每个元素为一个子句元组
